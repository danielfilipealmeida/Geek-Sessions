Node.js | 1

O que é?

É um ambiente de execução (runtime environent) implementado sobre o V8 (Google) que permite executar código Javascript na linha de comando.

É rápido, eficiente, controlado por eventos e com operações de Input e Output não bloqueantes.

Pensado para funcionar totalmente de forma assíncrona.

Node.js | 2

O que podemos fazer ?

  • Utilizar Javascript para programar servidores;
  • Operações I/O muito eficientes;
  • Acesso ao Sistema de Ficheiros e Bases de Dados de forma directa;
  • Utilizar o sistema de pacotes nativo (npm) para adicionar livrarias e extender as suas funcionalidades;

Express.js

Framework minimal para routing e que permite "crescer" através da utilização de "Middleware"

URL: http://expressjs.com

$ npm install express

Handlebars (1), HBS (2)

Livraria de Templating para Javascript (1) e View Engine para o Express(2).

Urls:

  • http://handlebarsjs.com
  • https://github.com/donpark/hbs

$ npm install hbs

Sequelize.js | 1

ORM para MySQL, PostgreSQL, MariaDB, Sqlite e Microsoft SQL Server baseado em Promises.

Urls:

  • http://docs.sequelizejs.com
  • https://github.com/sequelize/sequelize

$ npm install sequelize

Sequelize.js | 2

Algumas Features

  • Schema definition
  • Schema synchronization/dropping
  • 1:1, 1:M & N:M Associations
  • Promises
  • Transactions
  • Migrations
  • CLI (sequelize-cli)

Async.js | 1

Livraria de funções que facilitam "domar" a assincronia.

URL: https://github.com/caolan/async

$ npm install async

Async.js | 2

  • Versões async para map, reduce, filter, each
  • Outras patterns: series, parallel, waterfall
  • Seguem a convenção do Node.js em que o último argumento destas funções é um callback

Winston

Middleware de logging de informação para o express.

  • Níveis de Log
  • Formato de log configuravel, plain text, json
  • Um ou vários destinos de log
  • Rotação dos ficheiros de logs
  • Vários tipos destinos de logs (transports): console, ficheiros, http, Bases de dados

URL: https://github.com/winstonjs/winston

$ npm install winston

Chokidar

Wrapper que melhora as funções de watch de ficheiros do node.

  • Executa callbacks para determinados eventos sobre ficheiros: add, change, unlink, addDir, unlinkDir, etc.
  • Bastante configuravel: followSymlinks, persistent, ignore, interval, depth,...

URL: https://github.com/paulmillr/chokidar

$ npm install chokidar

Body-parser

Middleware do Express que permite extrair dados do body de uma chamada http ao nosso servidor.

Pode ser utilizado para retirar do body:

  • dados em Json
  • dados em formato texto
  • dados no formato Buffer do node.js. Pode conter imagens, videos, ficheiros comprimidos, etc.

URL: https://github.com/expressjs/body-parser

$ npm install body-parser

Busboy (1) e Connect-Busboy (2)

Modulo que permite fazer o parsing de dados oriundos de um formulário (1) e o middleware para express de ligação (2).

  • Parsing the forms multipart
  • Acesso aos ficheiros enviados no form

URL:

  • https://github.com/mscdex/busboy

  • https://www.npmjs.com/package/connect-busboy
$ npm install busboy

Connect-Flash

Armazena mensagens em memória de sessão. Utiliza-se para apresentar informações entre de uma página para outra.

Exemplos:

  • Mensagem de falha de autenticação
  • Mensagem de registo actualizado (ou não...)

URL: https://github.com/jaredhanson/connect-flash

$ npm install connect-flash

Express-Session

Middleware que implementa a funcionalidade de Sessões similiar ao PHP

Exemplos:

  • Mensagem de falha de autenticação
  • Mensagem de registo actualizado (ou não...)

URL: https://github.com/expressjs/session

$ npm install express-session
